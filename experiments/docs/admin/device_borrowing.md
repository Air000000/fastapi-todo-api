# 办公设备借用与归还流程

## 适用范围

本文适用于员工因临时工作需要借用公司设备的场景，包括备用笔记本、显示器、转接头、会议摄像头、麦克风、录音笔、测试手机、平板、投影设备、移动硬盘、演示器材和培训设备。长期配发的主力办公电脑不属于借用流程，由 IT 资产流程管理。借用设备的原则是用途明确、期限清楚、责任到人、按时归还、状态可追踪。

设备借用通常由行政和 IT 共同管理。行政负责会议、培训和通用办公设备；IT 负责电脑、测试终端、网络设备和涉及数据安全的硬件。员工不得在未登记的情况下直接从会议室、仓库或同事工位拿走设备。

## 可借用场景

常见可借用场景包括客户演示、临时出差、团队培训、公司活动、线上直播、设备故障期间的备用办公、兼容性测试和短期项目验证。借用申请应说明业务目的、使用地点、预计开始和结束时间、设备类型、数量和负责人。若设备将带离办公区，应特别注明。

个人用途、家庭娱乐、非公司项目、长期替代个人设备、未经批准的外部活动，不属于可借用范围。员工不得将借用设备转借给家人、朋友、客户、供应商或外包人员。外部人员如需使用公司设备，应由内部负责人申请并承担保管责任。

## 申请流程

员工应通过行政或 IT 服务门户提交借用申请。紧急情况可以先联系服务台，但仍需补录登记。申请通过后，负责人会确认设备库存、可借期限、领取地点和注意事项。领取时，员工应核对设备编号、配件、电源、线缆、外观状态和是否包含存储介质。

借用电脑、手机、平板或移动硬盘时，IT 可能要求完成额外安全检查，例如设备加密、账号登录、测试数据清理、软件版本确认和归还后擦除计划。涉及客户数据、生产数据或未公开资料的测试，必须使用批准的数据集和环境，不能把真实敏感数据复制到临时设备。

## 使用责任

借用人是设备第一责任人。使用期间应妥善保管，避免丢失、摔落、进水、过热或被未经授权人员使用。设备带离办公区时，应放入包内随身携带，不要留在无人看管的车辆、会议室、酒店大堂或客户现场。出差期间如需托运，应尽量避免托运包含数据的设备。

员工不得擅自拆机、刷机、重装系统、越狱、Root、卸载安全软件或修改资产标签。需要安装特殊软件时，应提前说明，由 IT 判断是否允许。演示或培训结束后，应退出个人账号、删除临时文件，并确认没有客户或员工资料遗留在设备中。

## 借用期限和延期

普通设备借用期限通常较短，应按申请中确认的日期归还。项目需要延长时，应在到期前提交延期申请，说明原因和新的归还日期。不要等行政或 IT 催收后才说明仍在使用。若设备库存紧张，延期可能被拒绝，员工应按期归还并寻找替代方案。

长期项目需要固定设备时，不应反复延期临时借用，而应走资产配置或项目采购流程。临时借用记录会用于库存和预算评估，频繁借用同类设备可能说明团队需要正式配置。

## 归还流程

归还时，员工应带回设备本体、充电器、线缆、保护套、说明卡、包装箱和其他配件。行政或 IT 会检查设备编号、外观、配件和基本功能。借用电脑、手机、移动硬盘等设备时，IT 会执行数据清理或恢复出厂设置。员工不应自行删除后就认为完成归还，必须等工作人员确认。

如果设备在归还时存在损坏、配件缺失或数据未清理，工作人员会记录状态并联系借用人补充说明。正常使用磨损通常不会追责，但因明显疏忽、违规使用或未按要求保管造成损失，可能需要部门或个人承担相应责任。

## 丢失和损坏处理

设备丢失、被盗或疑似被他人接触时，应立即通知 IT、行政和直属经理。包含公司账号、客户资料或内部文件的设备丢失，还应按安全事件流程处理。不要因为担心责任而延迟报告，越早报告越容易远程锁定、撤销会话和降低风险。

设备损坏时，应停止继续使用，避免扩大损坏。请说明发生场景、时间、地点和设备状态。不要私自找外部维修点维修公司设备，因为这可能导致保修失效、数据泄露或资产记录不一致。

## 数据和隐私要求

借用设备不应存放长期工作资料。演示材料、测试数据和录制文件应在使用后上传到公司批准的位置，并从设备中清理。不得在借用设备中登录个人网盘、私人邮箱或非必要社交账号。测试手机和平板可能会被多人轮流使用，更要避免保存个人照片、聊天记录或密码。

归还前，员工应主动退出所有账号，但不要绕过 IT 的标准清理。IT 清理是为了确保设备可安全再次使用，并不代表员工可以在使用期间随意保存敏感数据。

## 常见问题

**问题一：临时忘带电脑，可以借备用机办公一天吗？** 可以申请，但需说明用途和归还时间。备用机数量有限，不能长期替代个人主力设备。

**问题二：客户演示需要把设备借给客户操作，可以吗？** 可以在员工陪同和授权范围内使用，但不得让客户带离设备或访问无关资料。

**问题三：借用设备上的数据会被谁看到？** IT 会按安全流程清理设备，不以查看个人内容为目的。员工仍应避免在借用设备上保存私人信息或无关数据。## 边界场景与处理建议

如果设备用于跨部门活动，申请人应在申请中写明主责部门和现场负责人。活动当天可能有多人接触设备，但资产责任仍然归属于登记借用人。借用人可以指定现场协助人，但不能因此免除归还、清点和数据清理责任。大型活动结束后，建议在现场立即完成设备点数，避免第二天才发现配件遗失。

如果设备需要寄送到异地办公室、客户现场或员工住所，应使用公司认可的快递方式，并保留运单号。收到设备的一方应在当天确认外观和配件状态。运输途中损坏或丢失时，应由行政、IT 和申请部门共同判断责任，不应让接收人或寄出人单独处理。含有数据的设备原则上不使用普通快递寄送，确需寄送时应先完成加密和远程管理配置。

测试设备尤其需要注意账号隔离。测试手机、平板或浏览器环境可能被多个项目轮流使用，员工在登录测试账号时应确认环境属于当前项目，不要把真实客户资料、个人通讯录或私人照片导入测试设备。项目测试结束后，应将测试结论、截图和日志保存到项目空间，再清理设备本地内容。

## 文档维护

行政和 IT 每季度会根据库存、损耗、借用频率和员工反馈更新可借设备清单。员工如果发现设备长期不足、型号不匹配或借用流程影响交付，可以在服务门户提交建议。本文档由行政资产负责人维护，涉及安全要求的部分由 IT 安全团队共同复核。
